Accueil / Utilisation

Utilisation

Lancer le programme, comprendre les options de la ligne de commande et le format des fichiers de scène.

Ligne de commande

./raytracer [OPTIONS] [FICHIER_SCENE]

Sans fichier de scène, une scène vierge est chargée. Les options sont analysées par Options (srcs/config/Options.hpp) :

OptionDescription
-h, --helpAffiche l’aide et quitte.
-v, --versionAffiche la version (1.0.0) et quitte.
-r, --render [NOM].pngMode headless : rend la scène dans un PNG sans lancer l’interface. Le nom optionnel (finissant par .png) fixe la sortie (par défaut render.png).
--server [PORT]Lance l’interface et démarre un serveur de cluster. Le port optionnel fixe l’écoute (sinon automatique).
--connect IP PORTLance l’interface et rejoint le serveur de cluster à IP:PORT.

--render (headless) ne peut pas être combiné avec --server ni --connect, et --server et --connect sont mutuellement exclusifs.

Exemples de commandes

# Interface graphique sur une scène d'exemple
./raytracer tests/configs/subject.json

# Rendu direct vers un PNG (sans interface)
./raytracer -r output.png tests/configs/subject.json

# Interface + serveur de cluster sur le port 8080
./raytracer --server 8080 tests/configs/subject.json

# Interface + connexion à un serveur de cluster
./raytracer --connect 127.0.0.1 8080 tests/configs/subject.json

Mode headless (rendu PNG)

Le mode -r ne charge pas l’interface : il construit la scène, calcule l’image et l’exporte, puis quitte. Idéal pour les scripts et l’intégration continue.

./raytracer -r sortie.png tests/configs/cube.json
# Sortie attendue :
#   Loaded 2 renderer plugin(s)
#    - Viewport
#    - Default
#   Finished rendering scene! Saving to file..
#   Render has been saved to file 'sortie.png'

Format des fichiers de scène

Une scène est un fichier JSON lu par SceneParser. Il comporte quatre sections principales : environment, camera, objects et materials.

Squelette général

{
  "environment": { "ambient": 0.4, "diffuse": 0.6 },
  "camera": {
    "resolution": [1920, 1080],
    "position":   [0.0, -100.0, 20.0],
    "rotation":   [0.0, 0.0, 0.0],
    "fieldOfView": 72.0,
    "samplesPerPixel": 128
  },
  "objects": [
    { "name": "Sphere A", "type": "sphere", "position": [60, 5, 40], "radius": 25, "material": "Material A" }
  ],
  "materials": [
    { "name": "Material A", "model": "pbr", "base_color": [230, 60, 60], "roughness": 0.5 }
  ]
}

Section environment

CléTypeDéfautRôle
ambientnombre0.4Coefficient de lumière ambiante global.
diffusenombre0.6Coefficient de contribution diffuse global.

Section camera

CléTypeRôle
resolution[largeur, hauteur]Dimensions de l’image en pixels. Obligatoire.
position[x, y, z]Position de la caméra dans le monde.
rotation[x, y, z]Orientation (angles d’Euler, en degrés).
fieldOfViewnombreChamp de vision. Obligatoire.
samplesPerPixelentier > 0Échantillons par pixel (anti-aliasing). Défaut : 5. Alias : samples_per_pixel.

Section objects

Chaque objet possède un type (obligatoire) et un name. Les primitives peuvent référencer un material (par son nom). Le SceneParser ne lit que les clés pertinentes pour le type : les clés inconnues sont ignorées. Les types disponibles :

typeClés principales
sphereposition, radius, scale, material
planeaxis ("x"/"y"/"z"), position, size, material
cubeposition, rotation, scale, size, material
cylinderposition, rotation, radius, height, material
coneposition, rotation, radius, height, material
trianglevertex0, vertex1, vertex2, material
torusposition, rotation, radius, height, material
tanglecubeposition, rotation, size, threshold, material
fractalposition, size, power, iterations, material
meshfile (.obj) ou vertices/faces/normals, position, rotation, scale, vertex_overrides, material
point_lightposition, color, intensity
directional_lightdirection, color, intensity

Voir la page Fonctionnalités pour le détail de chaque primitive et de ses paramètres.

Couleurs et vecteurs

  • Un vecteur est un tableau [x, y, z].
  • Une couleur de lumière est un tableau [r, g, b] avec des composantes entières 0–255.

Section materials

Chaque matériau porte un name (référencé par les objets) et un model ("phong" ou "pbr"). Les principaux champs :

{
  "name": "Material A",
  "model": "pbr",
  "base_color": [230, 60, 60],
  "specular":   [255, 255, 255],
  "shininess": 32.0,
  "reflectivity": 0.0,
  "transparency": 0.0,
  "ior": 1.5,
  "metallic": 0.0,
  "roughness": 0.5,
  "ao": 1.0,
  "clearcoat": 0.0,
  "sheen": 0.0,
  "transmission": 0.0,
  "alpha": 1.0,
  "texture_map": "",  "texture_map_enabled": false,  "texture_uv_scale": 1.0,
  "normal_map": "",   "normal_map_enabled": false,   "normal_scale": 1.0
}

La liste complète des champs est décrite dans Fonctionnalités → Matériaux.

    {
      "name": "Sphere A",
      "type": "sphere",

De nombreuses scènes prêtes à l’emploi se trouvent dans tests/configs/ : cube.json, cone.json, torus.json, fractal.json, tanglecube.json, mesh_showcase.json, vertex_edit.json, etc.

Importer un maillage .obj

Un objet de type mesh peut charger un fichier .obj via la clé file. Les fichiers d’exemple sont dans tests/obj/.

{
  "name": "Cube",
  "type": "mesh",
  "file": "tests/obj/cube.obj",
  "position": [26.0, 10.0, 15.0],
  "rotation": [0.0, 0.0, 20.0],
  "scale": [14.0, 14.0, 14.0],
  "material": "Blue"
}

Un maillage peut aussi être défini en ligne (sans fichier) via vertices, faces et normals, ou porter des modifications de sommets via vertex_overrides — voir Édition de sommets.

Sauvegarde de scène

Depuis l’interface, la scène peut être ré-exportée en JSON : SceneRegister sérialise la caméra, l’environnement, les objets et les matériaux. Le cycle charger → éditer → sauvegarder → recharger restitue fidèlement la scène.
