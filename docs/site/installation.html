Accueil / Installation

Installation

Prérequis, dépendances et compilation du projet.

Prérequis

Le projet cible un environnement Linux (testé sous WSL2). Il faut :

Outil / bibliothèqueRôle
clang++Compilateur C++ (le Makefile l’invoque explicitement), standard C++17.
makeSystème de build.
SFML (≥ 2.5)Fenêtre, contexte graphique, réseau — utilisée par le plugin d’interface. Modules : graphics, window, system, network.
X11 (libX11)Système de fenêtrage sous Linux (lié via -lX11).

Dépendances déjà fournies

Deux bibliothèques d’en-têtes sont incluses dans srcs/external/ — rien à installer :

  • nlohmann/json (srcs/external/nlohmann/json.hpp) — lecture/écriture des fichiers de scène JSON.
  • stb_image (srcs/external/stb_image.h) — chargement d’images (textures) et export PNG.

Installer SFML et X11 (Debian/Ubuntu)

sudo apt update
sudo apt install clang make libsfml-dev libx11-dev

Les paquets s’appellent libsfml-dev et libx11-dev sous Debian/Ubuntu. Adaptez selon votre distribution.

Compilation

Depuis la racine du dépôt :

make

La compilation utilise les drapeaux -Wall -Wextra -Werror -std=c++17 -fsanitize=address (AddressSanitizer activé pour détecter les erreurs mémoire). Elle produit :

ArtefactDescription
./raytracerExécutable principal (cœur du moteur).
plugins/user_interface.soPlugin d’interface graphique (SFML).
plugins/renderer_default.soRenderer « Default » (rendu complet vers image).
plugins/renderer_viewport.soRenderer « Viewport » (rendu progressif interactif).

Les plugins sont chargés à l’exécution depuis le dossier ./plugins. Lancez donc ./raytracer depuis la racine du projet, là où se trouve ce dossier.

Cibles du Makefile

CommandeEffet
make / make allCompile l’exécutable et les trois plugins.
make cleanSupprime les fichiers objets .o.
make fcleanSupprime les .o, l’exécutable et les plugins .so.
make reÉquivalent à fclean puis all (recompilation complète).

Vérifier l’installation

# Version
./raytracer --version

# Aide (liste des options)
./raytracer --help

# Premier rendu vers un PNG
./raytracer -r sortie.png tests/configs/subject.json

Un fichier sortie.png doit apparaître. Passez à la page Utilisation pour la suite.

Sous WSL : l’interface graphique force le rendu OpenGL logiciel (Mesa/llvmpipe) pour rester stable avec plusieurs fenêtres. Un message Setting vertical sync not supported peut apparaître ; il est sans conséquence (voir la FAQ).
