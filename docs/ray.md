# Ray

## Definition

A ray is a half-line parameterized by scalar `t`:

```text
P(t) = O + t*D
```

- `O`: ray origin.
- `D`: ray direction (usually normalized).
- `t`: distance-like parameter along the ray.

## Fields

| Field | Type | Description |
|---|---|---|
| `origin` | vec3 | Start point `O` |
| `direction` | vec3 | Direction vector `D` |
| `t_min` | float | Lower valid bound for intersections |
| `t_max` | float | Upper valid bound for intersections |

## Usage in the Rendering Pipeline

### Primary Rays

Generated by the camera for each pixel to sample visible surfaces.

### Shadow Rays

Cast from shaded points toward lights to test visibility (occlusion).

### Secondary Rays (optional extensions)

Used in advanced effects:

- Reflection rays.
- Refraction rays.
- Indirect/global illumination rays.

## Practical Rules

- Keep `direction` normalized for stable shading and distance interpretation.
- Use `t_min = epsilon` for secondary rays to avoid self-intersections.
- Always test intersection candidates within `[t_min, t_max]`.
